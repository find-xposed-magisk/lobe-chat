# 交付检查与音频消息

## 新功能

- **任务交付检查。** 为任务添加检查项后，交付检查器会在结果交付前先做核验。某项检查未通过时，可以扣下结果并触发一轮修复，你还能看到一份核验报告，包含结论、评分以及每项检查背后的证据。
- **给助理发送音频。** 可以附带 MP3 等音频文件，Gemini 等支持音频的模型能够听到其中的内容；音频消息现在以波形播放器内嵌播放。
- **把文档转为技能。** 文档浏览器新增「转为技能」操作，可就地把任意助理文档变成可复用的技能。
- **CLI 自助升级。** 运行 `lh update` 即可把 LobeHub CLI 升级到最新版本，并可用 `file upload` 上传本地文件。

## 体验优化

- 创建助理时会根据场景从技能商店推荐相关技能。
- GLM-5.2 支持选择作答前的思考强度。
- 内置 Linear 结果以卡片呈现，不再是原始 JSON。
- 冷启动时改为一段连续的加载画面，不再出现品牌 Logo 闪烁。
- 含有未发送内容的话题会显示「草稿」提示，未读状态改由服务端同步，多设备间保持一致。
- 桌面端：用 Mod+1–9 快速跳转标签、用 Ctrl+Tab 循环切换，更新说明也会按 Markdown 渲染。
- 运行设备切换器新增「管理」入口，设备详情面板会列出最近使用的目录。
- 助理在已连接的聊天应用中失败时，回复会说明原因，而不是只给一串操作 ID。

## 问题修复

- 编辑连接器并保存时，不再悄悄清空已保存的凭据。
- 切换界面语言不再出现白屏闪烁。
- 添加自定义模型时会提前拒绝重复的模型 ID。
- 助理工作目录的清除按钮现在能真正清空。
- 搜索结果遇到空链接的引用时不再报错。
- 微信与 QQ 的文件附件（含音频）能更可靠地获取。
