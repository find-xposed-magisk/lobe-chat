# 更快的对话上下文与 Codex 额度

本周的更新集中在让进行中的对话更顺手。你可以找回最近输入、连同附件一起复用旧消息、把上下文交给另一个助理，也能在运行前看到本地编码助理还能使用多少额度。

## 新功能

- **用上下箭头找回输入历史。** 在输入框中按上 / 下箭头即可浏览最近输入，并通过预览选择器确认内容后再发送。
- **把富内容恢复到输入框。** 「恢复到输入框」会把之前的用户消息连同图片、文件一起带回输入框，方便你调整后再次发送，而不用手动重建。
- **把上下文转发给其他助理。** 选中多条消息后，可以把它们发送给另一个助理，让值得复用的上下文进入新的工作流。
- **把本地文件拖成路径节点。** 在已选择本地目录的模式下，拖入本地文件会生成可引用的路径节点，让助理知道要使用哪个本地文件。
- **运行前查看 Codex 额度。** Codex 控件现在会在输入栏展示剩余额度和重置次数，并与速度、worktree 控制一起服务本地编码会话。
- **查看助理级别用量。** 助理统计现在会展示单个助理的 Token、用量和费用数据，帮助你理解哪些助理带来了主要消耗。

## 体验优化

- **对话菜单更易扫描。** 输入菜单按区块分组，工作目录与 Git 控件更清晰，已完成的助理回合也会折叠成更干净的摘要。
- **助理文档更容易浏览。** 文档文件夹可以折叠为渐进索引行；缺失文档会显示明确的未找到状态，文档列表也会在编辑后刷新。
- **任务进度更容易判断。** 任务话题对话会在浮动面板中打开，运行卡片也会显示最新消息，无需逐个打开详情页即可了解进展。
- **审查与验收报告更清楚。** 审查面板现在提供双栏 diff 与文件树；验收报告会区分系统错误和未通过的交付检查。
- **供应商行为更可预期。** AiHubMix 可以使用配置的代理地址，Vertex 图片上传不再把 base64 当作文本泄露，供应商内容过滤错误也不会无意义重试。
- **技能与连接器界面更清楚。** 技能标签会展示描述并可打开对应的 `SKILL.md`，已连接的连接器详情会按连接状态展示，自定义 MCP 连接器也会出现在 @ 提及选择器中。

## 问题修复

- **对话创建与回复更稳定。** 话题创建中发送消息会正确排队，乐观创建的话题会留在正确的项目分组内，折叠回合后的最终回答也会保持可见。
- **工作区范围更严谨。** 首页最近项目、Skill Store 导入、群组助理移除、知识库文件访问和用户输入 URL 抓取，都会限定在当前工作区与调用者范围内。
- **桌面端与本地助理流程恢复更好。** 桌面端可以检测用户本地安装的 Claude Code，拖拽时保留本地路径，把人工介入回复路由到正确对话，并保持话题标签标题准确。
- **任务与文档边界问题已修复。** 缺失文档与任务状态不再卡住或崩溃，任务运行路由已修正，大文档读取也会限制长度以保护对话上下文。
- **助理运行失败更清楚。** 远程助理派发、过大的命令输出、缓存输入用量、工作流运行取消，以及网关模式下单个 `@agent` 路由的边界情况都得到处理。
- **安全加固。** URL 抓取、技能导入路径、知识库文件解析与工作区归属检查都更严格，降低跨范围访问风险。
